WriteOnSide (随边记) зберігає нотатки Markdown у вибраній вами теці. Немає приватної бази даних і обов’язкової хмарної служби, тож той самий Vault можна відкрити в WriteOnSide, Obsidian, VS Code або іншому редакторі.

> [!NOTE]
> WriteOnSide `0.0.50` — передрелізний проєкт у активній розробці. Зробіть резервні копії важливих нотаток і перегляньте примітки до випуску перед оновленням.

## Встановлення

### Завантажити збірку для Windows

Завантажте найновішу `WriteOnSide.exe` з
[GitHub Releases](https://github.com/Thymine2001/WriteOnSide/releases).

WriteOnSide зараз поширюється як портативна Windows-програма в одному файлі:

1. Завантажте `WriteOnSide.exe`.
2. Покладіть у теку, де хочете зберігати програму.
3. Запустіть і виберіть теку нотаток або наявний Vault Obsidian.
4. Використовуйте `Ctrl+Shift+Enter`, щоб показати або сховати панель.

Windows SmartScreen може показати попередження для непідписаних збірок розробки. Переконайтеся, що файл з цього репозиторію, перш ніж запускати.

## Основні можливості

### Бічна панель

- Панель без рамки на всю висоту, завжди поверх інших вікон
- Налаштовуване розташування зліва або справа біля краю екрана
- Глобальне сполучення показу/приховування, за замовчуванням `Ctrl+Shift+Enter`
- Регульована ширина панелі, ширина «Файли» та непрозорість
- Плавне відкриття, закриття, зміна макета та розміру
- Керування в системному треї та необов’язковий автозапуск Windows
- Одноекземплярна поведінка та позиціонування на робочих областях кількох моніторів

### Редагування Markdown

- Редаговане джерело Markdown із підсвічуванням синтаксису в реальному часі
- Режим лише для читання з посиланнями та зображеннями
- Панель інструментів для заголовків, наголосу, цитат, списків, завдань, таблиць, коду, посилань, зображень, виділення та кольору тексту
- YAML front matter із заголовками, тегами, датами, псевдонімами та іншими метаданими
- Пошук і заміна, номери рядків, структура та закріплені заголовки
- Блоки коду з міткою мови та копіюванням одним кліком
- Налаштовувані шрифти, розміри тексту, теми та сполучення команд

### Мови інтерфейсу

Програма містить **English**, **中文**, **Português**, **Deutsch**, **Français**, **Nederlands**, **한국어**, **Italiano**, **हिन्दी** та **Українська**. Змініть у **Налаштування → Загальні → Мова**.

### Сумісність з Obsidian

| Функція | Підтримка |
|---|---|
| Вікіпосилання: `[[Note]]`, `[[Note\|alias]]`, `[[Note#Heading]]` | Так |
| Посилання на блок: `[[Note#^block-id]]` | Так |
| Вбудовування нотаток і зображень: `![[file]]` | Так |
| Callouts: `> [!note]` | Режим читання |
| Виноски, `%%коментарі%%` та вбудовані `#tags` | Так |
| Списки завдань: `- [ ]` та `- [x]` | Так |
| Перейменувати нотатку та оновити вхідні wikilinks | Так |

Введення `[[` відкриває автодоповнення нотаток. `Ctrl+клацання` переходить за wikilink у режимі редагування. Інструмент зворотних посилань показує нотатки, що посилаються на поточну.

### Файли та вкладення

- Провідник файлів із відкладеним завантаженням і рекурсивним пошуком
- Фільтрація YAML-тегів із багаторазовим вибором
- Створення, перейменування, видалення, перетягування та попередній перегляд файлів
- Редагування Markdown і поширених текстових або кодових форматів
- Вставлення або перетягування зображень у нотатки
- Налаштовувана тека вкладень із переносними відносними посиланнями
- Атомарне збереження та резервні копії з міткою часу поза Vault
- Переглядач зображень із масштабуванням і прокручуванням

## Системні вимоги

**Для кінцевих користувачів:**

- Windows 10 або Windows 11
- Стандартний сеанс робочого столу; Windows on ARM ще не перевіряли офіційно

**Для розробки з вихідного коду:**

- Python 3.12
- Пакети з [`requirements.txt`](../requirements.txt)

## Запуск із вихідного коду

```powershell
git clone https://github.com/Thymine2001/WriteOnSide.git
cd WriteOnSide

py -3.12 -m venv .venv
.\.venv\Scripts\python.exe -m pip install --upgrade pip
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
.\.venv\Scripts\python.exe .\writeonside.py
```

Під час першого запуску виберіть теку нотаток або наявний Vault Obsidian.

## Базове використання

- Кнопка «гамбургер» відкриває або закриває провідник файлів.
- Перемикайте режими редагування та читання на панелі інструментів.
- Створюйте нотатки з панелі інструментів або провідника.
- Вставте зображення безпосередньо в нотатку Markdown, щоб скопіювати його в налаштовану теку вкладень.
- Виберіть YAML-теги в нижній частині провідника, щоб фільтрувати нотатки.
- Відкрийте Налаштування, щоб налаштувати теку нотаток, макет, ширини, непрозорість, тему, шрифт, глобальне сполучення та сполучення панелі інструментів.
- Закриття панелі ховає програму; використовуйте глобальне сполучення або меню трею, щоб показати знову.

## Тести

```powershell
.\.venv\Scripts\python.exe -m unittest discover -s tests -p "test_*.py"
```

Поточне дерево вихідного коду містить модульні тести для конфігурації, рендерингу Markdown, сполучень клавіш, сховища, індексації нотаток, синтаксису Obsidian, перейменування wikilinks та інтернаціоналізації.

## Збірка EXE для Windows

Скрипт випуску потребує PyInstaller і PyMuPDF на додаток до залежностей виконання:

```powershell
.\.venv\Scripts\python.exe -m pip install pyinstaller pymupdf
.\build_release.ps1
```

Скрипт:

1. Збільшує номер патча в `VERSION`.
2. Експортує PNG і ICO з SVG-логотипів.
3. Збирає однофайловий виконуваний файл Windows за допомогою PyInstaller.
4. Записує в `dist-native-tree-vX.Y.Z\WriteOnSide.exe`.
5. Зберігає три найновіші каталоги випусків.

Див. [`BUILDING.md`](../BUILDING.md) для додаткових команд і усунення несправностей.

## Розташування даних

| Дані | Розташування |
|---|---|
| Нотатки та вкладення | Тека, обрана користувачем, або Vault Obsidian |
| Налаштування | `%LOCALAPPDATA%\WriteOnSide\config.json` |
| Керовані резервні копії | `%LOCALAPPDATA%\WriteOnSide\Backups\` |

WriteOnSide не потребує облікового запису чи вбудованої хмарної служби. Нотатки залишають комп’ютер лише коли користувач розміщує Vault у окремо налаштованій службі синхронізації.
